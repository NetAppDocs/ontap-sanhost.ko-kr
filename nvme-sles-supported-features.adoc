= SUSE Linux Enterprise Server의 ONTAP 지원 및 기능에 대해 알아보세요.
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
NVMe over Fabrics(NVMe-oF)를 사용한 호스트 구성에서 지원되는 기능은 ONTAP 및 SUSE Linux Enterprise Server 버전에 따라 다릅니다.

[cols="50,25,25"]
|===
| 특징 | SUSE Linux Enterprise Server 호스트 버전 | ONTAP 버전 


| NVMe/TCP는 전송 계층 보안(TLS) 1.3 암호화를 지원합니다.  a| 
15 SP6 이상
| 9.16.1 이상 


| RHEL 호스트와 ONTAP 컨트롤러 간의 NVMe/TCP를 통해 안전한 인밴드 인증이 지원됩니다.  a| 
15 SP4 이상
| 9.12.1 이상 


| 고유한 검색 NQN을 사용하여 지속적 검색 컨트롤러(PDC)가 지원됩니다.  a| 
15 SP4 이상
| 9.11.1 이상 


| NVMe/TCP는 네이티브를 사용하여 네임스페이스를 제공합니다. `nvme-cli` 패키지  a| 
15 SP4 이상
| 9.10.1 이상 


| NVMe 및 SCSI 트래픽은 NVMe-oF 네임스페이스의 경우 NVMe 멀티패스를 사용하고 SCSI LUN의 경우 dm-multipath를 사용하여 동일한 호스트에서 지원됩니다.  a| 
15 SP1 이상
| 9.4 이상 
|===
ONTAP 시스템 설정에서 실행되는 ONTAP 버전에 관계없이 다음과 같은 SAN 호스트 기능을 지원합니다.

[cols="75,25"]
|===
| 특징 | SUSE Linux Enterprise Server 호스트 버전 


| NVMe/FC 프로토콜을 사용하여 SAN 부팅이 활성화됩니다.  a| 
15 SP7 이상



| 기본적으로 네이티브 NVMe 멀티패스가 활성화되어 있습니다.  a| 
15 SP1 이상



| 그만큼 `nvme-cli` 패키지에는 타사 스크립트가 필요 없는 자동 연결 스크립트가 포함되어 있습니다.  a| 
15 SP1 이상

|===
